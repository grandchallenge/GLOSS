# Release Checklist

## Correctness

- [ ] Stable toolchain build succeeds from a clean checkout.
- [ ] Unit, property, mutation, golden, and integration suites pass.
- [ ] Canonical output has no known silent quantifier, polarity, binder-domain, hypothesis, or relation errors.
- [ ] Unsupported constructs produce typed diagnostics and explicit unrendered nodes.
- [ ] Schema compatibility and migration tests pass.

## Reproducibility

- [ ] Toolchain, mathlib revision, dependency locks, corpus build, model identifiers, and seeds are recorded.
- [ ] Benchmark reports can be regenerated by documented commands.
- [ ] Generated artifacts carry hashes and provenance.

## Governance

- [ ] Required Council reviews are accepted.
- [ ] No blocking obligations remain.
- [ ] Terminology registry and decision records are current.
- [ ] Release notes distinguish behavior, schema, corpus, model, and interface changes.

## Product quality

- [ ] CLI exit codes and diagnostics conform to specification.
- [ ] Offline canonical mode works without network access.
- [ ] Accessibility checks cover pronunciation, punctuation, and screen-reader navigation.
- [ ] Security review covers untrusted Lean code, corpus licensing, prompt injection, and output escaping.

## Distribution

- [ ] Version numbers agree across package metadata, schema documents, and reports.
- [ ] SBOM and license notices are present.
- [ ] Checksums are generated.
- [ ] Stable and nightly compatibility status are reported separately.
